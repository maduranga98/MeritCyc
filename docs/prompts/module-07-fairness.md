# Module 7 — Fairness & Audit

## PROMPT 7.1 — Standalone Audit Trail Page

```
You are working on MeritCyc, a React 18 + TypeScript + Tailwind CSS + Firebase app.

Context:
- The CycleDetail page has a cycle-specific audit log tab.
- MISSING: a standalone, company-wide Audit Trail page showing ALL audit log entries across all cycles and all actions.
- Route: /audit-trail

Task:
Create src/pages/analytics/AuditTrail.tsx — a full audit trail page for HR admins and Super Admins.

Page requirements:

HEADER: "Audit Trail" title, "Export CSV" button (top right).

FILTERS BAR (horizontal, sticky below header):
- Date Range: "From" and "To" date pickers (default: last 30 days).
- Action Type: multi-select dropdown with grouped options:
  * User Lifecycle: USER_APPROVED, USER_REJECTED, USER_DEACTIVATED, USER_REACTIVATED, ROLE_CHANGED
  * Cycle Lifecycle: CYCLE_CREATED, CYCLE_PUBLISHED, CYCLE_CANCELLED, CYCLE_FINALIZED
  * Evaluations: EVALUATION_SUBMITTED, EVALUATION_DRAFT_SAVED, SCORE_OVERRIDDEN
  * Settings: COMPANY_SETTINGS_UPDATED, QR_REGENERATED, REGISTRATION_TOGGLED
- Actor: text search to filter by actor email.
- Target Type: select — user / cycle / evaluation / company / all.
- "Clear Filters" link.

AUDIT LOG TABLE:
- Columns: Timestamp | Actor | Action | Target | Details
- Timestamp: format "Apr 8, 2025 · 14:23" — full ISO on hover tooltip.
- Actor: actor email in slate-600.
- Action: colored badge chip based on action category (green for approvals, amber for changes, red for deletions/deactivations, blue for cycle events).
- Target: target type + target ID (truncated, copy-to-clipboard icon on hover).
- Details: `after` or `metadata` JSON — show as truncated key-value pairs, "View Details" link opens a modal with full JSON formatted nicely.
- Pagination: 50 per page, load-more or page navigation.

Data fetching:
- Query: `/auditLogs` collection where `companyId == user.companyId`, ordered by `createdAt desc`.
- Apply client-side filtering for action type, actor, target type.
- Date range filter applied as Firestore `startAt` / `endAt` on `createdAt`.
- Limit 500 documents maximum — show "Showing the most recent 500 entries. Use date filters to narrow results." if at limit.

Export CSV:
- When "Export CSV" is clicked, download the currently filtered results as a CSV.
- CSV columns: timestamp, actorEmail, actorRole, action, targetType, targetId, details (JSON stringified).
- Use Papaparse `unparse()` for CSV generation.

ProtectedRoute: allowedRoles ["hr_admin", "super_admin"].
Add route in App.tsx. Add "Audit Trail" link to the analytics section in the sidebar.
```

## PROMPT 7.2 — Pay Equity Report PDF Export

```
You are working on MeritCyc, a React 18 + TypeScript + Tailwind CSS + Firebase app.

Context:
- File: src/pages/analytics/FairnessDashboard.tsx
- The fairness dashboard exists but has no PDF export functionality.

Task:
Add a "Export Pay Equity Report" button to FairnessDashboard.tsx that generates and downloads a PDF.

Requirements:
1. Add an "Export Pay Equity Report" button in the top-right of the FairnessDashboard page. Style: border border-slate-300 text-slate-700 hover:bg-slate-50 flex items-center gap-2 with a Download icon.
2. On click: show a loading state "Generating PDF..." on the button.
3. Use jsPDF + html2canvas to generate the PDF:
   a. Create a hidden div (off-screen, not display:none) that renders the report content.
   b. Report content to render:
      - MeritCyc logo + company name as header
      - Report title: "Pay Equity Report — {companyName}" + generated date
      - Section 1: Fairness Score — the 0-100 score with a description of what it means
      - Section 2: Department Comparison Table — dept name, avg score, avg increment %, employee count
      - Section 3: Pay Gap Indicators — any flagged disparities
      - Section 4: Manager Scoring Consistency — table of managers and their scoring variance
      - Footer: "Generated by MeritCyc · Confidential · {date}"
   c. Use html2canvas to capture the hidden div.
   d. Add to jsPDF and save as "MeritCyc_Pay_Equity_Report_{YYYY-MM-DD}.pdf".
4. Use `useRef` on the hidden report div. Trigger html2canvas on the ref.
5. Show toast "Report downloaded" on success. Reset button state on complete.
6. The hidden report div should use Tailwind classes compatible with html2canvas (avoid CSS variables — use explicit color values).

Install check: jsPDF and html2canvas are already in the project (package.json has jspdf and html2canvas). Import from 'jspdf' and 'html2canvas'.
```
